AgentWatch

A macOS menu bar app that tells you which AI agents are running on your machine, what they're using, and lets you pause or kill them. This site hosts the public registry that powers the app's detection.

Registry

signatures.json

The detection rule set. 73 providers, 8 local-runtime endpoints, calendar-versioned. Fetched by AgentWatch on every launch.

signatures.schema.json

The JSON Schema (Draft 2020-12) the registry validates against. Use it if you're building tooling on top of the data.

How it works

AgentWatch detects AI processes using three clues, in order — first match wins:

  1. Process name match against the truncated kp_proc.p_comm (e.g. claude, aider, ollama).
  2. Argv keyword match for processes hosted in a generic runtime — node, python, VS Code's Code Helper.
  3. Network connection to a known localhost port or AI provider IP range.

The registry tells the app what to look for. Update the registry, every running AgentWatch picks up the change on its next launch.

Use it from your own tooling

curl -s https://raw.githubusercontent.com/medomar/agentwatch-public/main/signatures.json \
  | jq '.providers[] | select(.confidence == "high") | .id'

Contribute

Open a PR to add a new AI tool or fix an existing entry. CI validates every PR against the schema. See the contribution guide for the conventions (kebab-case ids, 16-char processNames cap, banned generic keywords like chat/gpt/llm).
